# FitGraph

Проект предназначен для моделирования ультрафиолетовых (УФЭ) и рентгеновских (РФЭ) фотоэлектронных спектров методом Гелиуса-Зигбана (DOI: https://doi.org/10.1039/DC9725400257), и интерпретации экспериментальных УФЭ и РФЭ спектров, полученных на источниках He I (21.2 эВ), He II (40.8 эВ) и Mg Kα (1253.6 эВ), Al Kα (1486.7 эВ). В скором времени проект будет расширен до произвольного выбора энергии ионизирующего излучения, что позволит найти его применение в синхротронных исследованиях.

Проект написан во фреймворке [Qt](https://www.qt.io/) для разработки кроссплатформенного программного обеспечения на языке программирования C++. В данном проекте используется сторонний виджет [qcustomplot.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/qcustomplot.cpp), взятый отсюда [QCustomPlot](https://www.qcustomplot.com/) для построения графиков и визуализации данных распространяемый по лицензии [GNU GPL](https://www.gnu.org/licenses/gpl-3.0.html).

**Научные** области: физика конденсированного состояния, материаловедение (нанотехнологии), квантовая химия, синхротронные исследования.
## Для кого?

Проект нацелен на широкую группу исследователей, занимающихся:
1.	Исследованием электронной структуры молекулярных систем;
2.	Эмиссионной фотоэлектронной спектроскопией (PES, UPS, XPS);
3.	Квантово-химическим моделированием молекулярных систем в пакетах программ [FireFly](http://classic.chem.msu.su/gran/gamess/), [ORCA](https://orcaforum.kofo.mpg.de/), [Gaussian](https://gaussian.com/), [GAMESS US](https://www.msg.chem.iastate.edu/gamess/) (в дальнейшем этот список расширится);
4.	Разработкой обменно-корреляционных функционалов для теории функционала плотности (DFT) и других квантово-химических методов моделирования;
5.	Созданием передовых материалов с заранее заданными свойствами;
6.	Созданием баз данных для обучения ИИ с целью создания новых материалов и прогнозирования их свойств.
## Зачем?

Кратко:

1.	Быстро сравнить расчётный спектр с экспериментальным;
2.	Быстрый и эффективный отбор квантово-химического метода моделирования (функционала, базиса) для исследуемых объектов;
3.	Оценка вклада отдельных атомов в спектральную полосу;
4.	Определение нужной конфигурации соединения путём сравнения спектров (теория/эксперимент) без использования рентгеноструктурного анализа.

Подробно:

Для всестороннего исследования электронной структуры молекулярных комплексов и новых материалов применяют экспериментальные методы: УФЭС, РФЭС, синхротронные исследования и другие области спектроскопии; а также методы компьютерного моделирования химических комплексов. В силу огромного количества расчётных методов необходимо удостовериться, что выбранная модель является подходящей в рамках данного исследования. Сравнение полос спектра с расчётными энергиями молекулярных орбиталей (МО) в приближении замороженных орбиталей является недостаточным по трём причинам. Во-первых, отсутствует информация об интенсивности расчётных полос, во-вторых, невозможно оценить вклад отдельно взятого атома в данную полосу. И в-третьих, интерпретация фотоэлектронных спектров молекулярных комплексов с количеством атомов >20 также становится затруднительной из-за большой плотности энергетических уровней, и отделить сигнал от шума не представляется возможным. Построение расчётного спектра вручную для сравнения с экспериментом является трудоёмкой задачей даже для опытных исследователей, поэтому многие этим пренебрегают, особенно если ряд исследуемых соединений состоит из большого количества комплексов. Автоматизация процесса моделирования спектров позволит значительно ускорить процесс отбора метода моделирования для ряда исследуемых соединений, а также быстрого сравнения теории и эксперимента, причём для систем с любым количеством атомов. 

Разработка программы для моделирования фотоэлектронных спектров с возможностью прямого сравнения теоретического и экспериментального спектра позволит решить эти проблемы. Актуальным также является вопрос верификации данных в научных публикациях в значимых отраслях при использовании вышеперечисленных методов исследования. При наличии данной программы и расчётных данных исследуемых объектов, проверить предоставленные результаты в научный журнал сможет любой желающий.
## Как работает?

Модель Гелиуса-Зигбана (приближение независимых центров) позволяет принять интегральную интенсивность (площадь) полосы как линейную комбинацию вероятности обнаружения (заселённости) электрона на атомном подуровне и его сечения фотоионизации с данного подуровня (вероятность, что при заданном световом потоке произойдёт вырывание этого электрона).

Класс [foutput.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/foutput.cpp) принимает на вход данные об электронной заселённости и энергии МО, из «.out-, .log-файлов» популярных пакетов вычислительной химии. [fdatapcs.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/fdatapcs.cpp) считывает значения атомных сечений фотоионизации и параметры асимметрии из [PhotoionizationCrossSection&Asymmetry.dat]( https://gitverse.ru/SkyThor/FitGraph/content/master/PhotoionizationCrossSection&Asymmetry.dat). [fcalculation.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/fcalculation.cpp) вычисляет сечения МО (интегральная площадь полосы) по модели Гелиуса-Зигбана и строит теоретический спектр, аппроксимируя каждую полосу псевдо-функцией Фойгта. Помимо спектра ПО наносит над энергетической шкалой вклады отдельных атомных подуровней в данную полосу спектра в виде цветных столбцов по цветовой схеме CPK. Класс главного окна программы [fitgraphmainwindow.cpp]( https://gitverse.ru/SkyThor/FitGraph/content/master/fitgraphmainwindow.cpp) содержит реализацию всех компонентов графического интерфейса пользователя. Класс [felement.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/felement.cpp) хранит данные о химических элементах, составляющие молекулу. [fexperimental.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/fexperimental.cpp) позволяет добавлять экспериментальный спектр для сравнения его с модельным. [fadjusting.cpp](https://gitverse.ru/SkyThor/FitGraph/content/master/fadjusting.cpp) в режиме сравнения позволяет сдвигать два спектра относительно друг друга, приравнивать интегральные площади обоих спектров, а также «подгонять» ширины всех полос автоматически под ширину экспериментальных пиков и перегибов.
## Дополнительные требования

Для использования **FitGraph** дополнительно необходимо иметь готовый «.out-, .log-файл» исследуемого комплекса, полученный в любой из следующих программ: [FireFly](http://classic.chem.msu.su/gran/gamess/), [ORCA](https://orcaforum.kofo.mpg.de/), [GAMESS US](https://www.msg.chem.iastate.edu/gamess/) (в дальнейшем, список расширится), а также (не обязательно) оцифрованный .dat-файл  экспериментального УФЭ и/или РФЭ спектра, полученный на любом из четырёх источников ионизирующего излучения: He I (21.2 эВ), He II (40.8 эВ) и Mg Kα (1253.6 эВ), Al Kα (1486.7 эВ).
## Как установить и запустить проект?

Чтобы настроить и запустить проект *Qt* с файлом *.pro*, вам понадобится интегрированная среда разработки *Qt Creator*. Далее пошаговое руководство:

### Шаг 1: Установка *Qt* и *Qt Creator*

Если у вас ещё не установлен *Qt*, вам нужно скачать и установить его.
1.	Перейдите на официальный сайт *[Qt](https://www.qt.io/)*;
2.	Скачайте *Qt Online Installer*;
3.	Во время установки выберите необходимые компоненты:

    1. ***Qt Creator*:** среда разработки;
    2.	**Компиляторы:** выберите подходящий компилятор для вашей операционной системы (например, *MinGW* для *Windows* или *GCC* для *Linux*);
    3.	**Версии *Qt*:** выберите одну или несколько версий фреймворка *Qt*.

### Шаг 2: Открытие проекта в *Qt Creator*

1.	Запустите *Qt Creator*;
2.	На главном экране или в меню выберите **Файл>Открыть файл или проект...**;
3.	Перейдите в папку вашего проекта и выберите *файл.pro*. *Qt Creator* автоматически распознает его как файл проекта;
4.	Нажмите **Открыть**.

### Шаг 3: Настройка комплекта (Kit)

После открытия проекта *Qt Creator* предложит вам настроить «комплект» (Kit). Комплект — это набор конфигураций, который определяет, какой компилятор, отладчик и версия Qt будут использоваться для сборки проекта. 
1.	*Qt Creator* покажет список доступных комплектов;
2.	Выберите один или несколько комплектов, которые подходят для вашего проекта. Обычно для десктопной разработки это *Kit* на базе *MinGW* или *MSVC* (*Windows*) или *GCC* (*Linux*);
3.	Нажмите **Настроить проект**.

### Шаг 4: Сборка проекта

После настройки комплекта *Qt Creator* загрузит проект. Теперь его можно скомпилировать.
1.	Нажмите на значок **Молоток** на панели слева или выберите **Сборка> Собрать проект**;
2.	Если проект собран успешно, в окне **Вывод сборки** появится соответствующее сообщение. Если возникнут ошибки, они будут отображены, и вам придётся их исправить.

### Шаг 5: Запуск проекта

Когда проект успешно скомпилирован, вы можете его запустить.
1.	Нажмите на зелёную кнопку **Запустить** на панели слева или выберите **Сборка> Запустить**;
2.	Если всё настроено правильно, приложение запустится. Если нет, убедитесь, что в конфигурации запуска выбран правильный исполняемый файл и комплект.
